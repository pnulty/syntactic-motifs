Literature Review: Syntactic Stylometry for LLM-Generated Text Detection
1. The Interpretability Gap in Computational Stylometry

Your abstract identifies a critical tension in the field. Traditional stylometric approaches based on frequency signatures of function words serve as an "uninterpretable impression of an author's style"—effective for grouping similar authors but fundamentally different from how literary scholars analyze style qualitatively.
Uga
This echoes a broader problem noted by researchers: computational stylometry has large potential for applications in literary science, forensics, and other areas, yet "many of the basic research questions of this field are not studied systematically."
ResearchGate
Researchers have argued for "more attention in research for explanation in computational stylometry as opposed to purely quantitative evaluation measures."
ResearchGate
The move toward syntactic features offers a promising path to more interpretable stylometry. Baayen et al. pioneered use of deep syntactic features by constructing "phrase structure rewrite rules" as tokens for stylometric analysis
Uga
—essentially treating grammar rules as a kind of vocabulary. More recent work has shown that features derived from Context Free Grammar (CFG) parse trees consistently improve detection performance over shallow lexico-syntactic features.
ResearchGate
2. Syntactic Features in Authorship Attribution

Several approaches to syntactic stylometry have emerged:

Phrase structure features: Early work used "counts and lengths for various types of phrases (noun phrases, verb phrases, and so forth)" with syntactic features outperforming purely lexical features.
Uga

k-embedded-edge subtrees: Kim et al. designed k-embedded-edge subtrees "with the intention of capturing syntactic information that reflects writers' natural tendencies to structure their sentences in particular ways." Unlike standard subtrees that only capture local relationships, these can "encode for longer-distance dependencies."
Harvard University
Dependency vs. constituency parsing: There's interesting evidence that "attribution based on dependency grammars performed significantly less well compared to attribution based on phrase structure grammars,"
Harvard University

though this may depend on the application.

Parse tree structure features: Recent work has proposed "novel text representation features created from parse tree structures... solely or primarily defined based on the tree structure" that "better reflect the pure structural properties of parse trees."
IEEE Xplore

These structural features were shown to complement existing syntactic features.
3. Frequent Subtree Mining: A Key Methodology

Your "syntactic motifs" idea aligns with a rich tradition in pattern mining. "In natural language processing (NLP), frequent subtrees mined from parse tree databases can be used to increase the accuracy of NLP tasks, such as sentiment analysis and question classification problems."
ResearchGate

The core algorithms include:

    TreeMiner (Zaki, 2002): Uses "string representations of trees for efficient tree manipulation and support counting" with depth-first search encoding.
    Wikipedia

Embedded vs. induced subtrees: Embedded subtrees allow "skipping the 'middle' node" to capture patterns that span non-contiguous tree nodes—important for capturing longer-range syntactic patterns.
Rensselaer Polytechnic Institute
HOPS algorithm: Uses "importance sampling to estimate the number of embeddings of a tree pattern" efficiently, with "runtime independent of target graph size."
ACM Digital Library
The challenge is computational: "Mining all frequent subtree patterns becomes infeasible for a large and dense tree database"
Wikipedia

due to combinatorial explosion.
4. LLM-Generated Text Detection: Current State

The detection literature reveals consistent stylistic signatures of LLM text:

Linguistic patterns: LLM-generated text exhibits "syntactical simplicity, use of passive voice, and narrativity," along with "enhanced organization, logical structure, formality, and objectivity." Linguistically, it "tends to be about twice the length of human-written text but exhibits a more limited vocabulary."
MIT Press

Register leveling: "The probabilistic nature of LLMs can blur the distinctions between writing styles and genres, resulting in what is known as register leveling"—where "distinct linguistic features characteristic of different genres or styles become less pronounced."
PubMed Central

Stylistic fingerprints: LLMs have "distinct and consistent stylistic fingerprints, even when prompted to write in different writing styles." These fingerprints are consistent "due to the deterministic nature of their training, fine-tuning, and text generation processes."
arXiv

Stylometric approaches: Recent work using StyloMetrix features with tree-based classifiers achieved up to 0.87 Matthews correlation coefficient in multiclass LLM identification scenarios.
arXiv

Mining Approach

Rather than mining all frequent subtrees (computationally prohibitive), consider:

    Template-based extraction: Define linguistically motivated "slots" to fill (e.g., "what fills the subject position of passive constructions?")
    Discriminative mining: Mine subtrees that discriminate between classes rather than just frequent ones—using techniques like discriminative pattern mining or contrast set mining.
    Depth-bounded motifs: Focus on subtrees of depth 2-4 from specific positions (S-level, VP-level) rather than arbitrary positions.
    Canonical form normalization: Abstract over lexical content while preserving structural skeleton (NP→DET ADJ N vs. NP→DET N, etc.)

    TreeMiner or custom implementation for pattern mining

Conversion Pipeline

# Constituency parsing → NetworkX graph
import spacy
import benepar
import networkx as nx
from nltk import Tree

nlp = spacy.load("en_core_web_sm")
nlp.add_pipe("benepar", config={"model": "benepar_en3"})

def tree_to_networkx(tree, graph=None, parent=None, node_id=0):
    """Convert NLTK Tree to NetworkX DiGraph"""
    if graph is None:
        graph = nx.DiGraph()
    
    current_id = node_id
    label = tree.label() if isinstance(tree, Tree) else str(tree)
    graph.add_node(current_id, label=label, is_terminal=not isinstance(tree, Tree))
    
    if parent is not None:
        graph.add_edge(parent, current_id)
    
    if isinstance(tree, Tree):
        for child in tree:
            node_id = tree_to_networkx(child, graph, current_id, node_id + 1)
    
    return node_id

doc = nlp("The cat sat on the mat.")
for sent in doc.sents:
    tree = Tree.fromstring(sent._.parse_string)
    G = nx.DiGraph()
    tree_to_networkx(tree, G)

A Taxonomy of Syntactic Motifs Aligned with Literary Stylistics

This is where it gets interesting. Here's a proposed taxonomy that bridges parse tree patterns with traditional rhetorical/stylistic categories:
1. Clause Complexity (Parataxis vs. Hypotaxis)

Parataxis "refers to the placing of two clauses next to one another without the use of subordinating conjunctions," while hypotaxis "refers to the arrangement of a sentence in which the main clause is built upon by phrases or subordinate clauses."
MasterClass

Syntactic signatures:

    Paratactic: S → S CONJ S (coordinate structures), S → S , S (asyndeton)
    Hypotactic: S → SBAR S, VP → VP SBAR, NP → NP SBAR (embedded clauses)

"Hypotaxis tends to increase syntactic complexity through greater embedding depth, which can demand more cognitive processing... Parataxis, however, emphasizes simplicity and additive progression."
Grokipedia

Motif features:

    Maximum embedding depth
    Ratio of SBAR nodes to S nodes
    Count of subordinating conjunctions (IN: because, although, when) vs. coordinating (CC: and, but, or)

2. Branching Direction

"In right-branching structures, typical of English, subordinate clauses follow the main clause... Left-branching languages place subordinates before the main clause."
Grokipedia

Motif features:

    Right-branching ratio (post-head modification)
    Left-branching depth (pre-head stacking, e.g., "the big red bouncy ball")
    Center-embedding frequency (self-embedding relative clauses)

3. Noun Phrase Complexity
Pattern	Literary Effect	Tree Signature
Heavy pre-modification	Dense, literary	NP → DT JJ JJ JJ NN
Heavy post-modification	Formal, academic	NP → NP PP PP
Nominalization	Abstract, bureaucratic	NP → DT NN (where NN ends in -tion, -ment)
Bare NPs	Stark, minimalist	NP → NN
4. Verb Phrase Patterns
Pattern	Literary Effect	Tree Signature
Passive voice	Formal, evasive	VP → VBN PP (by-phrase)
Progressive aspect	Immediacy	VP → VBG ...
Stative verbs	Descriptive	VP → VBZ ADJP
Serial verb constructions	Action-packed	VP → VP CC VP
Complement-heavy	Elaborate	VP → V SBAR
5. Sentence-Initial Structures

These are particularly diagnostic for style:
Pattern	Name	Tree Signature
Fronted adverbial	Periodic style	S → ADVP , NP VP
It-cleft	Focus/emphasis	S → NP[it] VP[be] NP SBAR
Existential there	Presentational	S → EX VP[be] NP
Participial opener	Literary	S → VP[VBG] , NP VP
Conjunctive opener	Additive	S → CC ...
6. Coordination Patterns

"Parataxis may use commas, semi-colons, and periods to force juxtaposition, but it can also replace these punctuation marks with 'and' to seamlessly string the speech or written piece together."
Wikipedia

Motif features:

    Polysyndeton: repeated CC (and...and...and)
    Asyndeton: coordinate structures without CC
    Tricolon: NP CC NP CC NP or VP CC VP CC VP

7. Rhetorical Figures with Syntactic Signatures
Figure	Definition	Detection Strategy
Anaphora	Repetition at clause start	Identical first-child patterns across siblings
Chiasmus	ABBA structure	Inverted tree patterns in adjacent clauses
Parallelism	Repeated structure	Isomorphic subtrees at same depth
Antithesis	Contrasting parallel	Parallel structure + contrastive lexis

from grandiso import find_motifs

def match_motif(motif_graph, sentence_graph):
    """Find all occurrences of motif in sentence parse tree"""
    # Use node_match to compare labels
    matches = find_motifs(
        motif_graph, 
        sentence_graph,
        node_match=lambda n1, n2: n1.get('label') == n2.get('label')
    )
    return matches

Phase 3: Aggregate to Stylistic Profile

def compute_stylistic_profile(documents):
    """Compute motif frequencies normalized by sentence count"""
    profile = defaultdict(float)
    
    for doc in documents:
        for sent in doc.sents:
            tree_graph = tree_to_networkx(sent._.parse_string)
            for motif_name, motif_def in MOTIFS.items():
                count = len(match_motif(motif_def['graph'], tree_graph))
                profile[motif_name] += count
        
        profile[motif_name] /= len(list(doc.sents))
    
    return dict(profile)
